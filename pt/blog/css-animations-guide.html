A ARTE DAS ANIMAÇÕES CSS

October 24, 2025 • 7 min read
csswebanimaçõesperformance

Criando animações suaves e com bom desempenho usando apenas CSS. Um mergulho profundo em como construir sistemas de movimento coerentes para a web moderna.

Por que Animações CSS?

As Animações CSS oferecem várias vantagens sobre as animações baseadas em JavaScript. Elas são aceleradas por hardware, mais eficientes e mantêm taxas de quadros suaves de 60fps mesmo em dispositivos de baixo custo. Mas, mais importante, elas possibilitam uma abordagem declarativa ao movimento — você define o que deve acontecer, e o navegador lida com o como.

Este blog é construído inteiramente com animações CSS-first. JavaScript é usado apenas para acionar alterações de estado; o comportamento visual reside nas folhas de estilo.

A Filosofia do Movimento

Antes de escolher as técnicas de animação, considere o que o movimento significa. Toda transição deve responder a três perguntas: De onde? Para onde? Como se relaciona com o que veio antes?

Neste site, as animações seguem três princípios fundamentais:
1. Continuidade - Tudo existe em uma superfície ininterrupta
2. Plausibilidade Física - O movimento ecoa a dinâmica do mundo real sem imitá-la
3. Energia Calma - Movimento com propósito, sem agitação

O movimento nunca é decoração. Ele estabelece relações espaciais e reconhece alterações de estado. A interface respira.

Os Blocos de Construção

CSS oferece três formas principais de criar animações:

Transições

Interpolações suaves entre estados. Usadas para efeitos de foco, mudanças de tema e expansões de menus suspensos. A chave é usar constantes de movimento compartilhadas. Todas as transições neste site usam a mesma curva de easing: cubic-bezier(0.4, 0, 0.2, 1).

:root {
    --motion-duration-core: 500ms;
    --motion-duration-quick: 300ms;
    --motion-duration-ripple: 600ms;
    --motion-easing: cubic-bezier(0.4, 0, 0.2, 1);
}

.custom-select {
    transition: all var(--motion-duration-core) var(--motion-easing);
}

Isso cria um ritmo temporal compartilhado. Toda interação parece parte da mesma coreografia.

Animações de Keyframe

Coreografia multi-etapas para movimentos complexos. Este blog usa keyframes com moderação — apenas para animações de carregamento inicial da página onde os elementos aparecem suavemente e sobem.

@keyframes fadeInUp {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

Propriedades de Transformação

A espinha dorsal do desempenho. translateY, scale e rotate são aceleradas por hardware e não vão acionar reflows caros. Quando o cabeçalho desliza para trás da barra de navegação, ele usa translateY(-120%) com uma duração de 500ms, combinando com a temporização principal do movimento.

API de View Transitions

Navegadores modernos suportam a API de View Transitions, que permite uma transição suave entre as navegações de página. Este blog a usa para transformar cards de posts em artigos completos:

::view-transition-old(post-container-1),
::view-transition-new(post-container-1) {
    animation-duration: var(--motion-duration-core);
    animation-timing-function: var(--motion-easing);
}

O ponto crucial: busque o conteúdo ANTES de chamar startViewTransition(). Mude o DOM de forma síncrona dentro do callback. Isso garante que o navegador capture os estados antigos e novos corretamente, prevenindo problemas de flash de conteúdo.

async function navigateTo(url) {
    const response = await fetch(url);
    const html = await response.text();

    document.startViewTransition(() => {
        // Troca síncrona do DOM aqui
        updateDOM(html);
    });
}

Desempenho Importa

Nem todas as propriedades CSS são criadas iguais quando se trata de desempenho da animação.

Aceleradas por hardware (GPU):
- transform (translate, scale, rotate)
- opacity
- filter (com cautela — pode ser custoso)

Evite animar:
- width, height, margin, padding (aciona o layout)
- color, background-color sozinho (aciona a pintura)
- box-shadow (custoso, mas aceitável para estados de foco sutis)

Use will-change com moderação e apenas durante animações ativas:

.post-card.filtering-out {
    opacity: 0;
    will-change: opacity;
}

.post-card:not(.filtering-out) {
    will-change: auto; /* Reinicia após a animação */
}

Isso diz ao navegador para otimizar a composição da GPU, mas apenas quando necessário. Deixar will-change ativo gasta memória.

O Ritmo da Interação

Todas as animações neste site seguem um ritmo temporal consistente definido por propriedades personalizadas CSS:
- 500ms (principal) para a maioria das transições (morfagem, deslizamento, expansão)
- 300ms (rápido) para microinterações (feedback de foco)
- 600ms (ondulação) para mudanças atmosféricas (transições de tema)
- Mesmo easing em todos os lugares: cubic-bezier(0.4, 0, 0.2, 1)

Essa consistência cria uma sensação unificada de "respiração". Quando os filtros expandem, menus suspensos abrem e cards se dissolvem, todos se movem no mesmo ritmo.

Revelações Escalonadas

Ao mostrar múltiplos elementos, escale sua aparição com atrasos de transição CSS:

.filter-row {
    transition: all var(--motion-duration-core) var(--motion-easing) 0.1s;
}

.filter-tags {
    transition: all var(--motion-duration-core) var(--motion-easing) 0.2s;
}

Os controles de filtro aparecem 100ms antes das pílulas de tags, criando uma cascata que guia o olhar para baixo naturalmente. Essa expansão orgânica faz com que a interface pareça estar se desdobrando de uma única superfície.

Limites Orgânicos

Um padrão único que este blog usa: ajuste de overflow. Quando uma lista suspensa se estende além do limite do painel de filtro, o bloco inteiro se expande da sua borda inferior, criando espaço organicamente:

.filters.expanded.dropdown-active {
    padding-bottom: 5rem;
}

JavaScript mede o transbordamento real do DOM usando getBoundingClientRect() e aplica a classe quando necessário. CSS lida com a expansão suave. O resultado parece que a página está ciente da sua própria geometria e responde fisicamente.

Transições de Tema

Alternar entre o modo claro e escuro não é apenas uma troca de cores. É uma mudança atmosférica de iluminação. A chave é mudar o tema PRIMEIRO, e depois habilitar as transições:

function applyTheme(theme, animated) {
    if (animated) {
        // Mudar o tema imediatamente
        document.documentElement.setAttribute('data-theme', theme);

        // Forçar reflow
        document.body.offsetHeight;

        // Habilitar transições suaves por 600ms
        document.body.classList.add('theme-transitioning');

        setTimeout(() => {
            document.body.classList.remove('theme-transitioning');
        }, 600);
    }
}

body.theme-transitioning * {
    transition: background-color var(--motion-duration-ripple) var(--motion-easing),
                color var(--motion-duration-ripple) var(--motion-easing),
                border-color var(--motion-duration-ripple) var(--motion-easing);
}

O efeito: a página inteira ondula para um novo estado de iluminação em 600ms, mantendo a ilusão de uma única superfície contínua. Sem flashes, sem estalos abruptos.

FLIP: Animando Alterações de Layout

Ao filtrar posts de blog, os cards precisam se reorganizar suavemente. A técnica FLIP (First, Last, Invert, Play) faz com que as alterações de layout pareçam animadas:

Coreografia de Duas Fases:
1. Dissolução: Cards perdendo relevância desaparecem suavemente em 500ms
2. Reorganização: Cards restantes deslizam para novas posições usando FLIP

// Fase 1: Capturar posições iniciais
const initialPositions = new Map();
cards.forEach(card => {
    const rect = card.getBoundingClientRect();
    initialPositions.set(card, { top: rect.top, left: rect.left });
});

// Aplicar mudanças de filtro (cards removidos do layout)
cardsToHide.forEach(card => card.classList.add('filtering-out'));

// Esperar a dissolução completar via evento transitionend
grid.addEventListener('transitionend', () => {
    // Fase 2: Capturar posições finais
    const finalPositions = new Map();
    visibleCards.forEach(card => {
        const rect = card.getBoundingClientRect();
        finalPositions.set(card, { top: rect.top, left: rect.left });
    });

    // FLIP: Calcular deltas e animar
    visibleCards.forEach(card => {
        const deltaY = initial.top - final.top;
        const deltaX = initial.left - final.left;

        card.style.setProperty('--flip-x', `${deltaX}px`);
        card.style.setProperty('--flip-y', `${deltaY}px`);
        card.classList.add('flip-animating');
    });
});

.post-card.flip-animating {
    transition: transform var(--motion-duration-core) var(--motion-easing);
    transform: translate(var(--flip-x, 0), var(--flip-y, 0));
    will-change: transform;
}

O resultado: cards se dissolvem, então a grade se contrai suavemente. Sem saltos bruscos. JavaScript controla o estado; CSS controla os visuais.

Exemplos Práticos

Toda interação neste blog demonstra esses princípios:

  • Foco do card: elevação translateY(-2px) com expansão da sombra (duração rápida de 300ms)
  • Pílulas de tags: Fundos gradientes com elevação de 3px, sombras em expansão (duração principal de 500ms)
  • Abertura de menu suspenso: max-height: 0 para 300px com desvanecimento da opacidade (500ms)
  • Expansão do filtro: Revelações escalonadas (atrasos de 100ms, 200ms) com crescimento pela borda inferior
  • Morfagem de posts: API de View Transitions para transformação de card para artigo (500ms)
  • Ondulação do tema: Todas as propriedades de cor fazem transição juntas (mudança atmosférica de 600ms)
  • Filtragem de cards: dissolução em duas fases (500ms) e depois reorganização FLIP (500ms)

Cada animação responde: De onde? Para onde? Como se relaciona? Nada aparece ou desaparece sem contexto espacial.

Acessibilidade Primeiro

Sempre respeite as preferências do usuário com a prefers-reduced-motion media query. Alguns usuários acham animações distrativas ou desorientadoras:

@media (prefers-reduced-motion: reduce) {
    * {
        animation-duration: 0.01ms !important;
        transition-duration: 0.01ms !important;
    }
}

O movimento deve aprimorar, nunca atrapalhar. Quando um usuário sinaliza que precisa de movimento reduzido, honre isso imediatamente.

A Conclusão

Animações CSS, quando abordadas sistematicamente, criam interfaces que parecem vivas, mas calmas. O objetivo não é impressionar com transições chamativas, mas construir confiança através da coerência espacial. O objetivo não era desempenho, mas ela deve também parecer com bom desempenho devido à ideia por trás das transições: fluidez. Uma transição instável tira a fluidez desta interface e, embora pudesse ser implementada em JavaScript puro, eu não sou (nem quero ser) um bom programador JavaScript. Usar intensivamente os recursos CSS para abstrair a lógica de implementação acelera o desenvolvimento, ao mesmo tempo que entrega as responsabilidades de otimização de desempenho para o próprio navegador.

Toda animação neste blog serve à mesma filosofia:
- Continuidade: Uma superfície ininterrupta
- Plausibilidade Física: Movimento que ecoa a dinâmica real
- Energia Calma: Com propósito, nunca agitada

Use propriedades personalizadas CSS para temporização compartilhada. Deixe o navegador lidar com o sequenciamento via eventos transitionend. Aplique will-change apenas durante animações ativas. Sempre pergunte: De onde? Para onde? Como se relaciona?

A interface não atua. Ela responde.
