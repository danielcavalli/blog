Construindo um Blog Bilíngue com View Transitions

Este blog é uma plataforma minimalista, com carregamento rápido, construída com geração de sites estáticos em Python, tradução automática via Gemini AI e transições de página fluidas usando a View Transitions API. Sem frameworks, sem dependências pesadas — apenas padrões web nativos fazendo o trabalho de verdade.

Visão Geral da Arquitetura

Static Site Generator: Um script Python (build.py) faz o parse dos posts em Markdown, extrai o frontmatter e gera HTML limpo. Rápido, simples, fácil de manter.

Tradução Automática: Cada post escrito em inglês é traduzido automaticamente para português do Brasil usando a Gemini 2.5 Flash API do Google. Um cache de tradução (`translation-cache.json`) usa hashes SHA-256 — as traduções só são regeradas quando a fonte em inglês muda.

View Transitions: A View Transitions API do Chromium cria transições contínuas e fluidas entre as páginas. Os cards do blog se expandem para artigos completos. A navegação parece uma reorganização de elementos em uma única superfície, e não saltos entre páginas.

Funcionalidades Principais

Geração de Conteúdo Bilíngue

translator = GeminiTranslator(api_key=os.getenv('GEMINI_API_KEY'))
translated = translator.translate_post(
    slug, frontmatter, content
)

O tradutor preserva anglicismos naturais (framework, backend, API), mantém os code blocks inalterados e produz português do Brasil fluente. Caching inteligente significa rebuilds rápidos — apenas o conteúdo alterado é retraduzido.

Navegação de Página Fluida

A View Transitions API permite transições fluidas entre as páginas:

  1. JavaScript intercepta cliques de navegação
  2. Novo HTML da página é obtido
  3. document.startViewTransition() captura estados antes/depois
  4. O browser anima o diff automaticamente

Todo o comportamento de animação é definido em CSS usando pseudo-elementos `::view-transition-*`. JavaScript apenas aciona a API.

::view-transition-old(blog-header) {
    animation: slide-header-up 500ms cubic-bezier(0.4, 0, 0.2, 1);
}

Design de UI Ambiente

O toggle de idioma mostra um ícone de globo com "EN / PT". O idioma ativo tem um **brilho ambiente suave** criado com gradientes radiais e filtros de blur:

.lang-en.active::before {
'';
    position: absolute;
    background: radial-gradient(circle, rgba(0, 123, 255, 0.12) 0%, transparent 70%);
    filter: blur(10px);
}

O toggle de tema usa um efeito ripple — transições de cor de 600ms em todos os elementos criam mudanças de iluminação atmosféricas. O objetivo: interação calma e contínua.

Filtragem de Posts com FLIP

A página de índice filtra posts por ano, mês e tags. A reorganização do layout usa a técnica FLIP (First, Last, Invert, Play):

  1. Captura as posições iniciais dos cards
  2. Aplica os filtros
  3. Captura as posições finais
  4. Anima os cards das posições antigas → novas

Animação em duas fases: dissolve cards ocultos (500ms), depois reorganiza cards visíveis (500ms). Continuidade espacial fluida sem saltos bruscos.

Sistema de Movimento

Todas as animações seguem constantes de tempo unificadas:

--motion-duration-core: 500ms;
--motion-duration-quick: 300ms;
--motion-duration-ripple: 600ms;
--motion-easing: cubic-bezier(0.4, 0, 0.2, 1);

Cada interação compartilha o mesmo ritmo. Os cards de posts levantam no hover (300ms), os headers do blog deslizam para fora (500ms), as mudanças de tema têm um efeito ripple (600ms). A consistência cria coesão.

Engenharia Anti-Flicker

Um desafio: prevenir o flicker visual durante View Transitions. A solução:

Remover classes CSS conflitantes antes de iniciar as transições:

// Clean up theme transition class before View Transition
document.body.classList.remove('theme-transitioning');
document.startViewTransition(() => {
    updateDOM(newContent);
});

A classe `theme-transitioning` aplica transições de cor de 600ms globalmente. Se ativa durante View Transitions, os dois sistemas de animação entram em conflito — flicker visível. A limpeza antes da navegação elimina isso.

Compatibilidade com Browser

Alvo: Apenas Chromium (Chrome, Edge, Brave, Arc). A View Transitions API é exclusiva do Chromium. Degradação graciosa: se a API não estiver disponível, a navegação funciona normalmente sem os efeitos de morphing.

Essa abordagem focada permite funcionalidades de ponta sem polyfills ou compromissos.

Por Que Essa Abordagem?

Dependências Mínimas: Sem React, Next.js ou framework bloat. APIs web nativas, Python, design cuidadoso.

Educacional: Cada linha é legível e documentada. O codebase ensina View Transitions, animações FLIP, CSS moderno.

Rápido: HTML estático com caching agressivo. Sem parsing pesado de JavaScript. Carregamentos quase instantâneos.

Acessível: Navegação por teclado, focus states, suporte a screen reader, HTML semântico em toda parte.

Manutenível: Script de build Python simples. Sem toolchains complexas.

Conclusão

O desenvolvimento web moderno pode ser simples, elegante, poderoso — sem frameworks pesados. Ao alavancar APIs nativas do browser (View Transitions), geração de sites estáticos e um design cuidadoso, criamos experiências polidas com código mínimo.

O design é calmo. A user experience é contínua. O código é mínimo. Esse é o objetivo: um blog que não atrapalhe e deixe o conteúdo brilhar.
